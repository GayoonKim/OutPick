# OutPick PRD

## 목적

OutPick의 제품 요구사항과 범위를 AI 에이전트가 빠르게 이해하기 위한 장기 기억 문서다.

## 제품 한줄 설명

확실하지 않음: 전체 제품 비전은 별도 기획 문서로 확정되지 않았다.

현재 코드 기준 OutPick은 로그인한 사용자가 프로필을 구성한 뒤 채팅과 룩북 기반 콘텐츠를 사용하는 iOS 앱으로 보인다.

## 현재 주요 기능 영역

### 로그인/부트스트랩

- 사용자는 Kakao/Google 등 소셜 인증 흐름으로 로그인한다.
- 기존 로그인 세션이 있으면 앱 시작 시 자동으로 인증 상태를 확인한다.
- 로그인 후 사용자 프로필이 없거나 불완전하면 프로필 플로우로 이동한다.
- 로그인 후 채팅/룩북 런타임 초기화를 수행한다.

### 프로필

- 사용자는 초기 프로필을 생성하거나 프로필 상세를 확인한다.
- 프로필 완료 여부는 메인 탭 진입 전 라우팅에 영향을 준다.

### 채팅

- 사용자는 채팅 목록과 참여 중인 채팅방 목록을 확인한다.
- 채팅방 진입, 생성, 편집, 검색, 설정, 미디어 조회 흐름이 존재한다.
- 채팅은 UIKit ViewController 기반 화면이 많다.

### 룩북

- 사용자는 브랜드, 시즌, 포스트 기반 룩북 콘텐츠를 탐색한다.
- 브랜드/시즌/포스트 상세 화면과 댓글/대댓글/신고/차단 흐름이 존재한다.
- 브랜드 또는 시즌 생성/가져오기 흐름이 존재한다.
- 좋아요/상호작용 상태는 앱 전역 store와 Repository/UseCase 흐름으로 관리한다.

### 좋아요 탭

- 사용자는 좋아요한 브랜드와 시즌을 `좋아요` 탭에서 확인한다.
- 장기 목표는 브랜드/시즌/포스트를 한 화면에서 섹션별로 보여주는 보관함 구조다.
- 현재 포스트 좋아요 섹션은 구현 진행 중이거나 placeholder 상태다.

### 마이페이지

- 사용자는 하단 탭에서 마이페이지에 진입한다.
- 확실하지 않음: 마이페이지의 상세 요구사항은 아직 이 문서에 정리되지 않았다.

## 제품 제약

- 요청 범위 밖 기능, 화면, 추상화는 추가하지 않는다.
- 새 기능은 기존 앱 라우팅, 탭 구조, MVVM-C + Repository + UseCase + DI 경계를 우선 따른다.
- Firebase/Firestore/Cloud Functions 접근은 UI 계층에서 직접 하지 않는다.
- 정책, 심사, 개인정보, 권한, 결제, UGC, AI, 외부 링크 리스크가 있으면 구현 전 사용자와 논의하고 공식 문서 확인 여부를 명시한다.

## 기능 설계 기준

새 기능 또는 큰 수정은 아래 4가지를 먼저 정리한다.

```text
요구사항:
구현 디테일:
제약 조건:
완료 기준:
```

모호한 부분이 있으면 구현하지 않고 사용자와 논의한다.

## 불확실한 부분

- 확실하지 않음: OutPick의 최종 제품 포지셔닝과 핵심 KPI는 아직 이 문서에 확정되어 있지 않다.
- 확실하지 않음: 마이페이지와 채팅의 제품 우선순위는 코드 구조 기준으로만 파악했다.
- 재확인 필요: 향후 기능별 PRD는 `docs/ai/features/` 또는 관련 task 문서에서 작성한 뒤 안정화된 내용만 이 문서에 반영한다.

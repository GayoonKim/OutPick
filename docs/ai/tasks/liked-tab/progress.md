# Liked Tab Progress

## 목적

좋아요 탭 작업의 현재 상태와 상세 진행 기록의 읽는 순서를 제공하는 인덱스다.

## 읽는 순서

- 현재 상태와 남은 작업: 이 문서.
- 상세 완료 기록과 변경 파일: `progress/completed.md`.
- phase 지도와 완료 기준: `plan.md`.
- 결정 인덱스: `decisions.md`.
- 결정 상세: `decisions/liked-tab.md`.

## 최종 목표

Lookbook의 좋아요 상태를 앱 전역에서 일관되게 공유하고, 사용자가 좋아요한 콘텐츠를 `좋아요` 탭에서 확인할 수 있게 한다.

좋아요 탭은 브랜드, 시즌, 포스트를 한 화면에서 섹션별로 보여주는 보관함 구조를 목표로 한다.

## 현재 상태

- Phase 1 현재 상태 재확인: 완료.
- Phase 2 좋아요 탭 섹션별 상태 분리: 완료.
- Phase 3 `AsyncLoadGate` 도입: 완료.
- Phase 4 좋아요 포스트 목록 연결: 완료.
- Phase 5 수동 QA와 커밋 정리: 일부 완료, 일부 남음.

## 완료 요약

- 하단 탭에 `좋아요` 탭을 추가했다.
- `LikedView`/`LikedViewModel`로 화면 이름을 일반화했다.
- 브랜드/시즌/포스트 좋아요 목록을 각각 user state 기준으로 페이지 조회하고 최신 문서를 합성한다.
- 브랜드/시즌/포스트 섹션은 독립 상태를 갖는다.
- 포스트 좋아요 목록은 2열 grid로 표시하고 기존 `PostDetailView`로 push한다.
- 브랜드/시즌/포스트 카드 모두 좋아요 취소 메뉴를 제공한다.
- 좋아요 취소는 optimistic remove 후 서버 실패 시 복구한다.
- 포스트 좋아요는 앱 실행 중 pull-to-refresh 없이 좋아요 탭에 즉시 반영된다.

## 남은 작업

- 브랜드/시즌/포스트 pagination 끝까지 스크롤 수동 QA.
- 섹션별 빈 상태, 실패 상태, 로딩 상태 시각 QA.
- 문서 변경 커밋 정리.
- `AsyncLoadGate`를 `LookbookHomeViewModel`까지 확장할지는 별도 범위로 분리한다.

## 불확실한 부분

- 재확인 필요: 인증 이후 화면 QA를 자동화하거나 테스트 계정으로 안정화할지 여부는 별도 논의가 필요하다.
- 재확인 필요: 원격에만 있는 `updateBrandLogoDetailPath(asia-northeast3)` 함수가 운영에서 필요한지.
- 확실하지 않음: 운영 데이터 규모에서 좋아요 브랜드/시즌/포스트 문서 재조회 방식이 충분히 빠른지는 실제 데이터로 확인하지 않았다.

# OutPick User Flow

## 목적

OutPick의 주요 사용자 흐름을 AI 에이전트가 구현 전에 확인하기 위한 문서다.

## 앱 시작 흐름

사용자 목표: 앱을 열고 인증 상태에 맞는 화면으로 진입한다.

진입 화면:

- iOS 앱 실행
- `SceneDelegate`
- `AppCoordinator`

주요 단계:

1. `SceneDelegate`가 `UIWindow`와 `AppCoordinator`를 준비한다.
2. `AppCoordinator`가 부트 로딩 화면을 표시한다.
3. `LoginManager`가 기존 로그인 상태를 확인한다.
4. 로그인 상태가 유효하면 사용자 프로필을 불러온다.
5. 프로필이 완성되어 있으면 메인 탭으로 진입한다.
6. 프로필이 없거나 불완전하면 프로필 플로우로 진입한다.
7. 로그인 상태가 없으면 로그인 화면으로 진입한다.

예외/빈 상태:

- 강제 로그아웃 이벤트가 오면 로그인 화면으로 루트 전환한다.
- 알림 route가 있으면 메인 탭 진입 후 가능한 시점에 처리한다.

완료 기준:

- 사용자가 인증 상태에 맞는 루트 화면에 도달한다.

## 로그인 흐름

사용자 목표: 소셜 로그인을 완료하고 앱을 사용할 수 있는 상태가 된다.

진입 화면:

- `LoginCompositionRoot`
- `LoginViewController`

주요 단계:

1. 사용자가 소셜 로그인을 시작한다.
2. Kakao 또는 Google URL callback이 `SceneDelegate`를 통해 처리된다.
3. 로그인 성공 시 `LoginManager`에 인증 사용자를 저장한다.
4. `AppCoordinator`가 프로필 확인 후 프로필 플로우 또는 메인 탭으로 라우팅한다.

예외/빈 상태:

- 로그인 실패, 토큰 교환 실패, 프로필 누락은 관련 ViewModel/Manager 흐름을 확인한다.

완료 기준:

- 인증된 사용자가 프로필 플로우 또는 메인 탭에 도달한다.

## 프로필 플로우

사용자 목표: 메인 앱 사용 전 필요한 프로필을 완료한다.

진입 화면:

- `ProfileCoordinator`
- `ProfileCompositionRoot`
- `UserProfileDetailCompositionRoot`

주요 단계:

1. 로그인 후 프로필이 없거나 불완전하면 프로필 플로우로 이동한다.
2. 사용자가 프로필 정보를 입력/수정한다.
3. 프로필 완료 후 채팅/룩북 bootstrapping을 수행한다.
4. 메인 탭으로 진입한다.

완료 기준:

- 프로필이 완료되어 메인 탭 진입이 가능하다.

## 메인 탭 흐름

사용자 목표: 앱의 주요 기능 영역으로 이동한다.

진입 화면:

- `MainTabCompositionRoot`
- `DefaultMainTabBuilder`
- `CustomTabBarViewController`

현재 탭 순서:

1. 채팅 목록
2. 참여 채팅방
3. 룩북
4. 좋아요
5. 마이페이지

완료 기준:

- 각 탭은 독립 root ViewController를 갖고, 탭 전환 시 해당 기능 root가 표시된다.

## 룩북 흐름

사용자 목표: 브랜드/시즌/포스트 콘텐츠를 탐색하고 상호작용한다.

진입 화면:

- `LookbookCompositionRoot.makeRoot`
- `LookbookHomeView`
- `LookbookCoordinator`

주요 단계:

1. 사용자가 룩북 탭에 진입한다.
2. 브랜드 목록 또는 룩북 홈 콘텐츠를 확인한다.
3. 브랜드 상세로 이동한다.
4. 시즌 상세로 이동한다.
5. 포스트 상세와 댓글/대댓글 흐름으로 이동한다.
6. 좋아요, 댓글, 신고, 차단 같은 상호작용을 수행한다.

완료 기준:

- 사용자가 룩북 콘텐츠를 탐색하고 주요 상호작용을 수행할 수 있다.

## 좋아요 탭 흐름

사용자 목표: 좋아요한 콘텐츠를 한 곳에서 확인한다.

진입 화면:

- `LookbookCompositionRoot.makeLikedRoot`
- `LikedView`
- `LikedViewModel`

주요 단계:

1. 사용자가 좋아요 탭에 진입한다.
2. 좋아요 브랜드 섹션을 확인한다.
3. 좋아요 시즌 섹션을 확인한다.
4. 포스트 섹션은 구현 예정 또는 placeholder 상태다.
5. 브랜드/시즌 카드를 눌러 상세 화면으로 이동한다.

완료 기준:

- 한 섹션 실패가 다른 섹션 표시를 막지 않는다.
- 사용자가 좋아요한 브랜드/시즌을 확인하고 상세로 이동할 수 있다.

## 채팅 흐름

사용자 목표: 채팅방 목록을 확인하고 방에 진입한다.

진입 화면:

- `ChatCompositionRoot`
- `ChatCoordinator`
- `ChatContainer`

주요 단계:

1. 사용자가 채팅 목록 또는 참여 채팅방 탭에 진입한다.
2. 채팅방 목록을 확인한다.
3. 채팅방을 생성/검색/편집하거나 기존 방에 진입한다.
4. 채팅 메시지, 첨부, 미디어, 설정 화면을 사용한다.

완료 기준:

- 사용자가 채팅방에 진입하고 기본 채팅 흐름을 수행할 수 있다.

## 룩북 콘텐츠 채팅방 공유 흐름

사용자 목표: 룩북에서 발견한 브랜드/시즌/포스트를 참여 중인 채팅방에 보내고, 필요하면 대화를 이어간다.

진입 화면:

- 브랜드 상세: `BrandDetailView`
- 시즌 상세: `SeasonDetailView`
- 포스트 상세: `PostDetailView`
- 공유 sheet: `LookbookShareSheetView` 후보
- 채팅방: `ChatViewController`

핵심 원칙:

- 채팅은 채팅답게 빠르게, 룩북은 들어갔을 때 정확하게.
- 채팅방에서는 snapshot 카드만 렌더링하고 원본 룩북 데이터는 조회하지 않는다.
- 원본 최신성/접근 가능 여부는 카드 탭 후 상세 화면에서 비동기로 확인한다.

주요 단계:

1. 사용자가 브랜드/시즌/포스트 상세에서 공유 버튼을 누른다.
2. 참여 중인 그룹 오픈채팅방 목록 sheet가 열린다.
3. 목록은 최근 메시지 시간 순으로 정렬되고, row는 방 대표 사진과 방 이름만 표시한다.
4. 사용자가 방 1개를 선택하면 전송 버튼이 활성화된다.
5. 사용자가 전송을 누르면 `chat:lookbookShare` 소켓 이벤트로 공유 메시지를 보낸다.
6. 전송 성공 후 sheet를 닫고 현재 룩북 화면에 완료 bar를 표시한다.
7. 사용자는 `이동`을 눌러 공유한 채팅방으로 가거나 `닫기`로 룩북 탐색을 계속한다.
8. 채팅방에서 공유 카드는 compact 카드로 표시된다.
9. 공유 카드를 탭하면 기존 룩북 상세 화면을 열고 snapshot을 먼저 표시한 뒤 원본을 비동기 조회한다.
10. 원본 조회 성공 시 최신 상세로 갱신하고, 삭제/접근 불가 시 볼 수 없는 콘텐츠 상태로 전환한다.

예외/빈 상태:

- 참여 중인 방 없음: `아직 참여 중인 채팅방이 없어요. 관심 가는 방에 참여한 뒤 공유해보세요.`
- 방 목록 로딩 실패: `채팅방을 불러오지 못했어요` + `다시 시도`
- 전송 실패: `공유하지 못했어요` + `다시 시도`
- 원본 삭제/권한 없음/접근 불가: `볼 수 없는 콘텐츠예요`
- 네트워크 실패: `콘텐츠를 불러오지 못했어요` + `다시 시도`

완료 기준:

- 공유 후 사용자를 강제로 채팅방으로 이동시키지 않는다.
- `이동` 선택 시 참여방 탭으로 전환한 뒤 해당 방을 열고, 뒤로가기는 채팅/참여방 탭을 유지한다.
- 채팅방 공유 카드 렌더링 경로에서 브랜드/시즌/포스트 원본 조회가 발생하지 않는다.

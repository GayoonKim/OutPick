# Implementation Document Size

## 목적

하네스 문서가 커지거나 책임이 섞일 때 압축, 인덱스화, 상세 문서 분리 기준을 정리한다.

## 적용 대상

- `ADR.md`, `PRD.md`, `DATA_SCHEMA.md`, `FLOW.md`, `SCREEN_SPEC.md`
- `architecture`, `entrypoints`, `workflows` 하위 문서
- task의 `plan.md`, `progress.md`, `decisions.md`와 장기 handoff 문서

## 자동 점검 시점

- 새 phase를 시작하거나 완료할 때.
- 한 문서에 결정, phase, feature, 책임 영역이 계속 추가될 때.
- 설계 문서에 구현 진행 로그, 검증 결과, 운영 배포 기록, smoke QA 결과가 함께 쌓이기 시작할 때.
- 같은 문서에서 필요한 내용을 찾기 위해 여러 섹션을 반복 탐색하게 될 때.
- 문서 갱신 후 약 300줄을 넘겼을 때. 줄 수는 분리 명령이 아니라 탐색 비용을 재검토하는 신호로 사용한다.

## 정리 순서

1. 오래된 진행 로그, 완료된 phase의 세부 과정, 중복된 설명을 현재 상태 중심으로 압축한다.
2. 다른 문서와 중복된 내용은 한 곳을 기준 문서로 정하고 나머지는 링크와 짧은 요약만 남긴다.
3. 압축 후에도 책임 영역이 둘 이상 섞이거나 약 300줄 이상이면 주제, phase, domain 단위 상세 파일로 분리한다.
4. 기존 파일은 삭제하거나 단순 이름 변경하지 않고, 목적과 읽는 순서를 제공하는 인덱스로 유지한다.
5. 분리한 모든 상세 문서가 인덱스와 관련 entrypoint에서 발견 가능한지 확인한다.

## 인덱스에 반드시 남길 내용

- 이 문서 묶음의 목적.
- 처음 읽을 파일.
- 질문이나 작업 유형별로 볼 상세 파일.
- 현재 상태와 다음 작업을 볼 파일.
- 결정 번호, ADR 번호, schema 항목처럼 기존 참조를 찾는 방법.

## 분리 기준 예시

- `design.md` 또는 `{phase}-design.md`: 최종 설계 요약과 상세 링크만 유지하고, phase 진행/검증/배포 기록은 `progress/{phase}.md`로 이동한다. 선택 이유와 대안은 `decisions/{phase-or-domain}.md`로 이동한다.
- `plan.md`: 전체 phase 지도와 현재 상태만 유지하고, phase별 긴 완료 기준이나 세부 실행 순서는 `plan/{phase}.md` 또는 관련 design 문서로 분리한다.
- `decisions.md`: 결정 요약과 상태만 유지하고 `decisions/{phase-or-domain}.md`로 상세 분리.
- `ADR.md`: ADR 목록과 짧은 요약을 유지하고 `adr/ADR-NNN-title.md`로 상세 분리.
- `DATA_SCHEMA.md`: 공통 원칙을 유지하고 `data-schema/{domain}.md`로 feature별 schema 분리.
- `architecture/*.md`: 전체 책임 경계를 유지하고 독립 subsystem은 하위 상세 문서로 분리.
- `progress.md`: 현재 목표, 최근 완료, 남은 작업만 유지하고 오래된 phase 기록은 `history/`로 이동하거나 압축.

## 원칙

- 파일 개수를 줄이는 것보다 필요한 문서만 선택해서 읽을 수 있게 만드는 것을 우선한다.
- 작은 문서를 기계적으로 잘게 나누지 않는다. 독립된 질문에 답하거나 반복 참조되는 단위일 때만 분리한다.
- 상세 문서에 같은 배경 설명을 반복하지 않고 인덱스 또는 공통 문서에 연결한다.
- 분리 과정에서 결정의 이유, 대안, 트레이드오프, 재검토 조건을 삭제하지 않는다.
- 완료된 phase의 세부 기록은 삭제하지 않고 `progress/`와 `decisions/`로 이동한 뒤 인덱스에서 발견 가능하게 둔다.
- 링크, 문서 지도, `HANDOFF.md`, task 포인터가 깨지지 않았는지 `rg`로 기존 경로 참조를 확인한다.
- 하네스 크기 점검에서 정리가 필요하다고 판단되면 다음 구현 phase로 넘어가기 전에 정리 후보와 구조를 사용자에게 보고한다.
